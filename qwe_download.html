Tutorial

Work flow

  1. Upload files and different combination of files means different methods:
    • Upload phenotype file + pedigree file + genotype file: sinle-step genomic best linear unbiased prediction (ssGBLUP)
    • Upload phenotype file + pedigree file : best linear unbiased prediction (BLUP)
    • Upload phenotype file + genotype file : genomic best linear unbiased prediction (GBLUP)
    • Upload variance component file + Any of the file combinations above: predict estimated breeding values without variance components estimation (fastest)
  2. Specify model components based on column names in uploaded phenotype file. Please note that only single-trait analysis is supported at present version, although multiple traits can be specified. When multiple traits are specified, the traits will be analysed one by one using the same model.
  3. Check the model you specified at the left bottom.
  4. Submit your email address and wait the results.

File format instruction

  1. Phenotype file:
    • Phenotype file can be comma or white space separated
    • Phenotype file MUST have column names (header)
    • The first column of phenotype file MUST be animal ID
    • Other columns can be other variables that may be needed in the model
    • The missing value is also allowed and can be indicated as NA
    • An example of phenotype file is below:
  2. Pedigree file:
    • Pedigree file can be comma or white space separated
    • Pedigree file MUST have column names (header)
    • The first column of phenotype file MUST be animal ID; the second column MUST be the sire of individuals in the first column; the third column MUST be the dam of individuals in the first column; the fourth column MUST be the order variables, such as birth date or other numberic values that can be used to sort the pedigree
    • The missing parent is allowed and can be indicated as NA
    • An example of pedigree file is below:
  3. Genotype file:
    The genotype files were in Plink bed format, and the standard format can be found here . Besides going by basic rules of Plink, the fam file is required to be prepared using the following rules:
    • No header is allowed
    • The first and second columns of fam file MUST both be animal ID
    • An example of fam file is below:
  4. Variance component file:
    • No header is allowed
    • Each row represents the variance comonents for each trait, and each column represents the variance components for each random effect across different traits
    • An example of variance component file is below, corresponding to the model containing three random effect in total, animal effect, dam effect, and random residual effect for two traits:
